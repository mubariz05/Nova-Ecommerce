# Nova Commerce - Contribution Guide

## Git Workflow

### Branch Naming

Create a new branch for every task.

Examples:

feature/homepage

feature/plp

feature/pdp

feature/cart

feature/account

feature/checkout

Never work directly on **main**.

---

## Commit Messages

Use Conventional Commits.

Examples:

feat: add hero banner

feat: create product card

fix: cart badge not updating

fix: responsive navbar

refactor: improve folder structure

---

## Pull Requests

After completing your task:

- Push your branch
- Open a Pull Request
- At least one teammate must review it
- Merge only after approval

---

## Merge Conflicts

If a merge conflict occurs:

1. Pull latest main
2. Merge main into your branch
3. Resolve conflicts
4. Test the project
5. Push again
6. Update the Pull Request
